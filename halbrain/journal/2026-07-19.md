### [16:00] Runtime crash on TestFlight Build 1 → env-var fix, Build 2 (branch: feature/ios-codemagic-ci · commits: a7204a8, 4d9598a · builds #6 installed, #7 green)
**Synopsis:** Build 1 installed and launched from TestFlight (whole pipeline proven end-to-end) but showed "Application error: a client-side exception has occurred while loading localhost." Diagnosed and fixed the runtime config, plus cleared the export-compliance gate. Build 2 (build #7) rebuilt green with the fix and is in Apple processing.
**What worked:**
1. **Export compliance** — TestFlight Build 1 sat in "Missing Compliance." User answered the encryption question (exempt) to unblock Build 1; I added `ITSAppUsesNonExemptEncryption=false` to `apps/web/ios/App/App/Info.plist` (committed, not cap-sync-overwritten) so future builds skip it. Matches SaleSonar.
2. **The crash: missing NEXT_PUBLIC_* at build time.** `supabase-browser.ts` calls `createBrowserClient(process.env.NEXT_PUBLIC_SUPABASE_URL!, ..._ANON_KEY!)` which throws on undefined; the mobile layout invokes it on mount. Next inlines `NEXT_PUBLIC_*` at build time, but the Codemagic clone has no `.env.local` (gitignored) so they compiled to undefined → throw on first paint. "loading localhost" is just Capacitor's local origin, a red herring. Grepped `apps/web/src` for `process.env.NEXT_PUBLIC_*`: only SUPABASE_URL/ANON_KEY (crash-critical, no fallback), APP_URL (has `|| https://helloagainlinks.com` fallback so non-crashing but must be correct), plus MOBILE_AUTH_CALLBACK_URL/API_IMPORT_ENABLED (fallbacks, skipped) and NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY (NOT referenced in client src at all — skipped). Fix: created a Codemagic **environment variable group `hal_mobile_env`** (values stored in Codemagic, NOT git — respects the credential-leak caution in [[reference_credentials]]) holding NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY, and NEXT_PUBLIC_APP_URL=https://helloagainlinks.com (overriding the localhost:3001 in `.env.local` — its own comment says prod uses the domain). Referenced `groups: [hal_mobile_env]` from both workflows in `codemagic.yaml`.
Evidence: build #7 all steps green incl. "Build .ipa for App Store Connect" (1m15s real archive+sign), status → publishing/upload to App Store Connect on commit 4d9598a. Apple ID confirmed: phone's Media & Purchases = rob@thespeciales.com = the internal tester, so no Apple-ID mismatch (the earlier "Redeem" screen was just the pre-compliance window; user had in fact installed Build 1).
